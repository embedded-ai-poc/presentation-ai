병렬 Agent
실행

여러 관점에서 동시에 분석

왜 병렬인가

순차 실행

1명이 3가지 관점
→ 편향 발생
→ 시간 오래 걸림

병렬 실행

3명이 각자 1관점
→ 독립적 분석
→ 빠른 완료

병렬 Agent 패턴

# 예시: 코드 검증 단계 병렬로 4개 Agent 실행: Agent 1: 요구사항 검증자 → 요구사항 매칭 여부 확인 Agent 2: 코드 품질 검사자 → MISRA C 준수, 복잡도 체크 Agent 3: 테스트 감사자 → 커버리지, 엣지 케이스 확인 Agent 4: 보안 검토자 → 취약점, 메모리 이슈 체크

Task 도구로 병렬 실행

# Claude Code 프롬프트 "다음 4개 Agent를 병렬로 실행해줘: 1. 요구사항-검증자: PRD와 코드 매칭 확인 2. 품질-검사자: MISRA C 2012 준수 확인 3. 테스트-감사자: 커버리지 80% 확인 4. 보안-검토자: 메모리 이슈 확인 각 Agent 결과를 종합해서 보고해줘."

워크플로우 패턴 1: 검증 루프

# 3단계 검증 워크플로우 Phase 1: 요구사항 검증 → PRD 분석 → 요구사항 추출 → 검증 Phase 2: 설계 검증 → 아키텍처 분석 → 인터페이스 검증 Phase 3: 코드 검증 → 구현 → 테스트 → 리뷰 각 Phase에서 실패 시 해당 단계 재시도 최대 3회 실패 시 사람에게 에스컬레이션

워크플로우 패턴 2: 병렬 검증

# 다중 관점 병렬 검증 검증 시작 ├─ Agent 1: 기능 검증 ├─ Agent 2: 성능 검증 ├─ Agent 3: 보안 검증 └─ Agent 4: 코드 품질 검증 모든 Agent 완료 후 → 결과 종합 → 신뢰도 점수 계산 → 최종 판정

워크플로우 패턴 3: 루프백

# 실패 시 자동 복구 실행 → 테스트 → 실패? ├─ Yes → 원인 분석 → 수정 → 재실행 │ (최대 3회) │ └─ No → 다음 단계 3회 실패 시: → 로그 수집 → 사람에게 알림 → 대기
