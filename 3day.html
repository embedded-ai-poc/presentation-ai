Skills

워크플로우를 패키지로 만들기

/명령어 하나로 복잡한 작업 실행

Skill이란

Skill 폴더 구조

~/.claude/skills/my-workflow/ ├── SKILL.md # 스킬 정의 (필수) ├── templates/ # 템플릿 파일들 │ └── prd.md ├── prompts/ # 프롬프트 모음 │ └── analysis.md └── examples/ # 예시 파일들

SKILL.md만 있으면 스킬로 인식됩니다

SKILL.md 기본 구조

--- name: my-workflow description: 코드 리뷰 자동화 워크플로우 triggers: - /review - /코드리뷰 --- # 코드 리뷰 워크플로우 ## 실행 단계 1. 변경된 파일 목록 확인 2. 각 파일 코드 리뷰 3. 리뷰 결과 정리

실제 Skill 예시: 코드리뷰

--- name: code-review description: MISRA C 기준 코드 리뷰 triggers: [/review, /리뷰] --- # 코드 리뷰 워크플로우 ## 검토 기준 - MISRA C 2012 위반 사항 - 메모리 누수 가능성 - NULL 포인터 체크 누락 ## 출력 형식 | 파일 | 라인 | 심각도 | 내용 |

Skill 실행하기

# 터미널에서 $ claude # Claude Code 내에서 > /review src/gc_module.c # 결과 SKILL.md의 워크플로우대로 자동 실행

슬래시 명령어로 간단하게 실행

Commands
& Hooks

자동화의 핵심 요소

Commands: 커스텀 명령어

# settings.json { "customCommands": { "/build": "make -j8 TARGET=release", "/test": "make test SUITE=unit", "/lint": "cppcheck --enable=all src/" } }

자주 쓰는 명령어를 단축키처럼 등록

Hooks: 자동 트리거

이벤트 발생
→
Hook 실행
→
자동 동작

특정 이벤트에 자동으로 반응

Hook 종류

PreToolUse

도구 실행 전

PostToolUse

도구 실행 후

Stop

세션 종료 시

SubagentStop

서브에이전트 종료

Hook 설정 예시

# settings.local.json { "hooks": { "PostToolUse": [{ "matcher": "Write", "hooks": [{ "type": "command", "command": "make lint" }] }] } }

파일 저장 후 자동으로 lint 실행

Hook 활용 사례

  • 코드 작성 후 — 자동 포맷팅, lint 실행
  • 테스트 실행 후 — 커버리지 리포트 생성
  • 세션 종료 시 — 작업 로그 저장
  • 서브에이전트 종료 — 결과 집계, 다음 단계 트리거

Context7 활용

라이브러리 문서를 AI 맥락에 주입

# 프롬프트 예시 "Context7으로 FreeRTOS 문서를 참조해서 태스크 생성 코드 작성해줘." # AI가 자동으로 1. Context7 MCP 호출 2. FreeRTOS 문서 검색 3. 최신 API 기반 코드 생성

사내용 MCP 서버 구축

# 가능한 커스텀 MCP - 사내 위키 검색 MCP - JIRA 이슈 조회 MCP - 코드 리뷰 시스템 연동 MCP - 빌드 서버 연동 MCP # 장점 AI가 사내 시스템과 직접 연동 → 컨텍스트 스위칭 최소화

MCP 보안 고려사항

  • 인증 정보 — 환경변수로 관리, 코드에 하드코딩 금지
  • 권한 최소화 — 읽기 전용으로 시작
  • 로깅 — MCP 호출 기록 남기기
  • 사내망 한정 — 외부 노출 차단

MCP 트러블슈팅

  • MCP 서버 연결 실패 — npx 경로 확인, 방화벽 체크
  • Context7 응답 없음 — API 키 환경변수 확인
  • 권한 오류 — 실행 권한 부여 (chmod +x)
  • 로그 확인 — ~/.claude/logs/ 디렉토리

문제 발생 시 claude mcp status 명령으로 상태 확인

병렬 Agent
실행

여러 관점에서 동시에 분석

왜 병렬인가

순차 실행

1명이 3가지 관점
→ 편향 발생
→ 시간 오래 걸림

병렬 실행

3명이 각자 1관점
→ 독립적 분석
→ 빠른 완료

병렬 Agent 패턴

# 예시: 코드 검증 단계 병렬로 4개 Agent 실행: Agent 1: 요구사항 검증자 → 요구사항 매칭 여부 확인 Agent 2: 코드 품질 검사자 → MISRA C 준수, 복잡도 체크 Agent 3: 테스트 감사자 → 커버리지, 엣지 케이스 확인 Agent 4: 보안 검토자 → 취약점, 메모리 이슈 체크

Task 도구로 병렬 실행

# Claude Code 프롬프트 "다음 4개 Agent를 병렬로 실행해줘: 1. 요구사항-검증자: PRD와 코드 매칭 확인 2. 품질-검사자: MISRA C 2012 준수 확인 3. 테스트-감사자: 커버리지 80% 확인 4. 보안-검토자: 메모리 이슈 확인 각 Agent 결과를 종합해서 보고해줘."

각 Layer의 특성에 맞는 검증 워크플로우

워크플로우 패턴 1: 검증 루프

# 3단계 검증 워크플로우 Phase 1: 요구사항 검증 → PRD 분석 → 요구사항 추출 → 검증 Phase 2: 설계 검증 → 아키텍처 분석 → 인터페이스 검증 Phase 3: 코드 검증 → 구현 → 테스트 → 리뷰 각 Phase에서 실패 시 해당 단계 재시도 최대 3회 실패 시 사람에게 에스컬레이션

워크플로우 패턴 2: 병렬 검증

# 다중 관점 병렬 검증 검증 시작 ├─ Agent 1: 기능 검증 ├─ Agent 2: 성능 검증 ├─ Agent 3: 보안 검증 └─ Agent 4: 코드 품질 검증 모든 Agent 완료 후 → 결과 종합 → 신뢰도 점수 계산 → 최종 판정

워크플로우 패턴 3: 루프백

# 실패 시 자동 복구 실행 → 테스트 → 실패? ├─ Yes → 원인 분석 → 수정 → 재실행 │ (최대 3회) │ └─ No → 다음 단계 3회 실패 시: → 로그 수집 → 사람에게 알림 → 대기
